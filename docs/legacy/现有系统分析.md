AndyClaw 投资系统设计研究报告（v1）

https://github.com/Andy-HNU/AndyClaw

有一个市场收益分析理论视频 原文意义不大可以直接看下面的结论。

结论先放前面：一个视频最多提供两条好原则——“时间优势”和“期望值思维”——但不足以单独构成完整系统。 你真正需要的系统，不应建立在“猜市场”或“比对手盘更聪明”上，而应建立在 目标、约束、资产配置、行为约束、复盘、学习 这六个闭环上。你现有 repo 已经有长期框架、配置、月投、再平衡、数据源降级这些骨架，但还缺 IPS/治理层、目标求解器、基准与概率指标、决策权限层、学习引擎。

一、先说总判断

你的视频里最有价值的两点，确实能进系统：
第一是 时间优势。第二是 用赔率/期望值替代“精准预测”。但视频里那种“市场主要是零和对手盘博弈”“拼信息和速度”的叙事，只适合提醒你不要自大，不适合作为你系统的主逻辑。你的 repo 也已经写明了：长期框架优先，不鼓励短线交易，不提供高频、杠杆、追涨杀跌型建议；自动化主要用于记录、计算、预警和汇总，而不是替你高频下判断。

外部证据也支持这种改写。Vanguard 把长期纪律拆成四个动作：定期投入、在波动里保持持有、再平衡、定期回顾；SEC 也明确提醒，波动时不要做 rash decisions，不要搞 all in / all out，很多时候更合理的是小幅调整而不是方向性翻转。对个人投资者来说，真正稳定的优势不是更快、更早，而是 更少无效交易、更多坚持规则。

二、你当前系统已经有了什么

从仓库看，你已经写下了非常明确的长期框架：长期年化目标 8%–10%、投资周期 10 年以上、核心方法是 永久组合思想 + 资产配置优先 + 指数投资为核心 + 产业 ETF 增强收益；目标配置是 股票 50%、债券 25%、黄金 15%、现金 10%；结构上又分成 防御层 / 市场层 / 产业增强层；月度新增资金按 12000 计划投入，优先补低配；再平衡采用 年检 + 偏离超过 ±10% 的双触发。数据侧也已经有主源/备源/快照过旧提示的降级策略。

这套骨架其实已经很接近“可运行系统”了。问题不在于你没有规则，而在于规则还停留在 配置层，没有真正上升到 治理层和决策层。当前 PROJECT 的一级目标仍然是仓位管理、风险预警、周报月报、新闻收集与辅助建议；而你更在乎的“学习投资”“跟住市场动态”“知道目标能不能达成”，还没有被提升成系统一级对象。performance ledger 目前也只有一个 baseline 和一条 entry，asset research 还是空数组，所以你才会强烈感觉“零件不少，但拼图没齐”。

三、完整系统需要哪些理论支柱
1）先有治理，再有判断：IPS 思路

CFA 的 IPS（Investment Policy Statement）框架很适合你现在的阶段。它要求先写清：谁负责定政策、谁负责执行、谁负责监控、多久复审、风险怎么量、报告怎么做、再平衡边界是什么。它还明确要求写出投资目标、回报要求、风险要求、约束条件、评估周期，以及 rebalancing to target allocations 的流程。对你来说，这一步不是形式主义，而是把“长期配置脑”和“盘中情绪脑”分开。

这意味着你的系统最上面不该是“今日建议”，而该是一个 Personal IPS / 投资宪法：写清楚目标、期限、现金流、最大可接受回撤、允许资产、禁止行为、评估窗口、复审节奏。没有这层，后面的巡检、预期、买卖、复盘都会互相打架。

2）先定义成功，再讨论收益：Goals-Based

你现在最关键的升级，不是“把预期收益算得更玄”，而是把系统目标从 “追一个年化数” 改成 “提高在给定风险下达成目标的概率”。Goals-Based Wealth Management 的研究明确提出：对个人投资者而言，风险更自然的定义不是组合波动率，而是 达不到目标的概率。这类框架强调，系统要围绕目标、时间、资金缺口和可承受失败概率来设计，而不是只围绕 benchmark 或短期收益来设计。

这正好和你的诉求一致。你说自己想做的是“大概率、小收益、正预期”，那你的目标函数就不该是 max expected return，而应该是：在不触发难以承受回撤的前提下，最大化 10 年目标的达成概率。 这也是为什么我不建议把视频里的“赔率裂缝”直接翻成事件下注，而建议你把它翻译成 目标调整后的期望值。

3）资产配置、分散和再平衡仍然是主轴

SEC/Investor.gov 的基本原则非常适合给你的系统打底：资产配置要围绕 时间期限和风险承受能力 来定；分散既要在大类资产之间做，也要在资产内部做；窄主题基金或 ETF 可能根本不是真分散，所以要检查 top holdings 是否重叠；再平衡的作用不是追求收益最大化，而是让组合重新回到与你目标相符的风险水平，而且通常 相对低频 更合适。

Vanguard 的研究与此一致：配置要匹配 willingness 和 ability to take risk，要接受 expected return 与 volatility 的 tradeoff，要广泛分散，并在目标和现实变化时定期复查。它还特别强调，不再平衡的组合会逐步漂向更高权益暴露，可能在未来遭遇更大的 drawdown。

所以，你当前的 50/25/15/10 + 三层结构 不是明显错误，反而方向上是成立的；真正的问题是：这个配置 是否足以支持你 8%–10% 的年化目标，以及你股票 50% 里面的产业增强层 是否被严格预算、严格限权。这要靠目标求解器和重叠度/集中度约束来回答，而不是靠感觉。

4）为什么核心要更像指数，卫星要更像卫星

你的 repo 已经写明“指数投资为核心，产业 ETF 增强收益”。这个方向有很强的外部支持。Vanguard 的研究显示：长期里，成本越低，投资者留在手里的回报越多；在其假设案例里，0.1% 与 2.0% 年成本差异，30 年后会造成非常大的终值差距。它也明确写道，低成本基金历史上在扣除成本后往往优于高成本基金。

S&P 的 SPIVA 和 Persistence Scorecard 则给了另一个角度：在 2025 年，美国大盘主动基金里有 79% 跑输 S&P 500；而过去表现好的主动基金，未来持续跑赢的比例也很低，2022 年末跑赢的大盘基金，到 2024 年还能继续跑赢的比例只有很小一部分。换句话说，主动胜率不高，持续胜率更低。这并不意味着卫星仓没意义，但意味着系统设计上，核心应该默认低成本、分散、难出错；卫星才承担增强收益的尝试，而且必须被硬预算。

5）行为防火墙不是附属件，是主发动机

如果只选一篇最适合写进你系统说明书的论文，我会把 Barber & Odean 放进去。它研究了 66,465 个家庭账户，结论非常直接：交易越频繁，净收益越差；高换手群体年化回报明显落后；作者的中心结论就是 “trading is hazardous to your wealth”。这跟你视频里“别总以为自己比对面聪明”其实是同一条逻辑。

Vanguard 的 behavior gap 研究则进一步说明：很多投资者损失的不是“市场没给回报”，而是自己在波动里偏离了原来的长期配置。其文中给出的一个定义下，2000–2012 年平均 annual shortfall 约 1.55 个百分点。而更稳定地持有、按目标配置行事，可以显著减少这种损失。

这意味着你的系统必须内置 行为防火墙：
不是天天问“买不买、卖不卖”，而是先判断“今天有没有权限动”。
不是用情绪触发交易，而是用规则触发交易。
不是因为难受就卖，而是因为 目标变了、权重漂了、假设失效了、或更高 EV 的替代方案出现了 才动。
这部分不是可选插件，而是系统最核心的风险控制。

6）市场要看，但不是为了做大宏观赌注

你第二个诉求是“跟住市场动态，学习投资”。这件事也需要系统化。AQR 的研究指出，不同资产对 增长、通胀、实际利率、波动率、流动性 等宏观因子的敏感度不同；做组合时应该先 map 出组合的宏观风险暴露和集中点，再做分散，但要 避免基于宏观预测做过大的战术押注。

这非常适合你的系统。也就是说，你需要一个 市场状态层，但它的职责不是“预测明天涨跌”，而是：
今天更像哪种环境？
我的组合对这种环境暴露了什么？
风险集中在哪里？
当前结论只能影响“解释、观察和小幅微调”，不能直接变成激进换仓。

7）学习也要有证据，不要靠“多看资讯”

你有知识库、遗忘曲线库，这是非常好的方向，但现在还没有被投资系统正式吸收。学习科学里，APS 对 10 种常见学习方法的综述把 practice testing 和 distributed practice 评为高效方法；Bjork 的经典研究也指出，retrieval practice 会提升后续提取能力，而 expanding intervals 对长期保持尤其有效。

这对你的系统的启发很直接：
学习市场，不该是“每天看更多新闻”，而该是把与你持仓最相关的知识做成 可复习、可提问、可回忆 的卡片。
也就是说，你的遗忘曲线库不该只记术语，而要记：资产角色、风险概念、错误案例、宏观关系、决策教训。

四、我建议的系统结构
1. 治理层：Personal IPS

这是系统最高层，写清：

目标金额或目标生活水平

投资期限

最大可承受回撤

最低现金储备

允许资产、禁止资产

谁能改规则、多久复审一次

何种情况允许改配置，何种情况只能观察

这样做的原因，是把“系统规则”和“当日情绪”彻底分开。CFA 的 IPS 框架就是这么要求的。

2. 目标层：Goal Solver

这层不预测市场，专门回答：
在当前资产、每月投入、目标期限、可承受风险下，当前配置有多大概率达成目标？

你的系统现在有目标，但没有求解器。因为 Vanguard 和 Goals-Based 研究都强调：没有任何配置能保证结果，所以 8%–10% 应当被当成 待验证假设，而不是先验真理。

3. 配置层：Strategic Allocation + Hard Budgets

这层保留你现在的大类资产框架，但把规则写得更硬：

核心层：默认指数和广泛分散

防御层：债券/黄金/现金，负责生存和缓冲

卫星层：产业增强，但必须明确预算上限、单主题上限、重叠约束

原因很简单：SEC 已经提醒过，窄主题基金即便数量不少，也可能并不分散；你的 repo 也已明确增强层不能破坏组合稳定性。

4. 决策层：Goal-Adjusted Expected Value

这层才是把视频真正吸收进去的地方。
但这里的 EV 不是“赌一个事件”，而是：

动作分数 = 对目标达成概率的改善
－ 交易成本
－ 集中度惩罚
－ 回撤惩罚
－ 行为失控惩罚

也就是说，你系统真正该算的不是“明天涨不涨”，而是“做这件事，比不做，更能提高我达到目标的概率吗？” 这比单纯看收益率更符合你的长期目标。

5. 巡检层：Inspection / Market State

每天可以巡检，但不能每天改结构。
日巡检只回答：

数据可靠不可靠

仓位有没有漂

卫星有没有过度集中

市场更像哪种宏观状态

今天只允许观察、补低配、再平衡、还是风控

这层的设计依据，一半来自 SEC/Vanguard 的再平衡原则，一半来自 AQR 的宏观暴露思路。

6. 执行层：Slow Rules, Tight Permissions

我建议把执行权限固定成四类：

观察

用新增资金补低配

再平衡

风险收缩 / 假设失效处理

而且要把节奏拉开：

日度：只巡检，不做结构性判断

月度：用新增资金修正低配

年度或阈值：再平衡

滚动 4–8 季度：评价系统成败

这和 CFA 对 evaluation horizon 的建议、你 repo 里的月投和 ±10% 再平衡、以及 Vanguard 对 yearly review/discipline 的建议是相容的。

7. 学习层：Learning Map + Retrieval

这层把“看市场”变成“学市场”：

资产卡：股票/债券/黄金/现金各自为什么存在

风险卡：回撤、久期、估值、通胀、相关性、集中度

关系卡：增长/通胀变化对各资产意味着什么

错误卡：最近一次情绪化操作、错因、下次如何识别

然后用你的遗忘曲线库做 expanding retrieval。
这样学习就不再是一盘散沙，而是直接服务组合。

五、为什么这套设计比“看视频做交易”更稳

因为它把视频里最好的两点保住了，又把最危险的两点剔除了。

保住的是：

时间优势

期望值思维

仓位管理

自我认知/行为约束

剔除的是：

把投资理解成主要靠对手盘输给你

把市场理解成主要靠短期赔率裂缝和速度优势赚钱

你的系统如果想长期有效，最该赚的不是“新闻差”“速度差”，而是这三种钱：

价值创造的钱：来自核心宽基和生产性资产长期增长。

风险补偿的钱：来自在可承受范围内承担股票、债券、通胀和流动性风险。

情绪/再平衡的钱：来自别人失衡时，你按规则用新资金和再平衡慢慢修正价格偏离。
这三类，比视频里前半段那种“和高频拼桌”的逻辑，更适合你。

六、对你 repo 的具体改造建议

建议在 system/ 下继续加 6 个文件：

11_personal_ips.md：投资宪法

12_goal_solver.json：目标求解器

13_benchmark_model.json：战略基准、行为基准、无操作基准

14_decision_policy.md：什么情况下允许哪类动作

15_decision_card.template.md：每次动作必须先填卡

16_market_state.md：增长/通胀/流动性状态

17_learning_map.md：学习主线与复习计划

18_review_scorecard.json：绩效、规则遵守、学习完成度三张分数表

同时在工作流里新增一个统一入口：
inspection → goal check → allocation drift → decision permission → action/no action → review → learning card。
这会把你现在“仓位 / 风险 / 报告 / 新闻”的底座，升级成真正的个人投资操作系统。

七、这份研究最终给你的答案

你的系统应该这样定义：

一个以目标达成概率为核心、以资产配置为底座、以低成本指数核心为主、以小预算卫星增强为辅、以行为防火墙限制冲动、以再平衡替代预测、以学习闭环提升认知的长期投资系统。

为什么这样设计？
因为外部证据反复指向同一个结论：
个人投资者最稳定的优势，不是更快，不是更会猜，不是更会找新闻；而是 目标清晰、分散、低成本、低换手、纪律、复盘和学习。

参考文献与证据来源

最核心的参考文献，我建议你把下面这些当作系统级文献来保存：

CFA Institute，《Elements of an Investment Policy Statement for Individual Investors》：用来定义治理、职责、复审、风险、约束、报告和再平衡边界。

Investor.gov，《Asset Allocation and Diversification》：用来定义时间期限、风险承受力、分散和低频再平衡。

Vanguard，《Principles for Investing Success》：用来支撑 goals / balance / cost / discipline 四原则，以及定投、持有、再平衡、定期复查。

Vanguard，《Investors winning as a behavior gap shrinks》：用来证明行为偏差会显著吞噬长期回报。

Barber & Odean，《Trading Is Hazardous to Your Wealth》：用来支撑“少交易、少自信过度、少以为自己比市场聪明”。

Das et al., 《A New Approach to Goals-Based Wealth Management》：用来把风险定义成“达不到目标的概率”，而不是只看波动率。

S&P SPIVA Year-End 2025 与 Persistence Scorecard：用来支撑“指数核心优先、主动增强受限、不要迷信持续跑赢”。

AQR，《Exploring Macroeconomic Sensitivities》：用来支撑“做市场状态巡检，但避免大宏观赌注”。

APS / Dunlosky，《Improving Students’ Learning With Effective Learning Techniques》与 Bjork，《Retrieval Practice and the Maintenance of Knowledge》：用来支撑遗忘曲线库、复习卡和学习闭环。

你自己的 repo 文档：PROJECT、RULES、00–05、10、performance_ledger、asset_research template，用来界定现有系统边界与缺口。
